- Running `/security-scan` or `/audit` (security dimension)
- Any changes to auth, middleware, or file upload handlers
- Before any production release
- A new module is added with approval or financial workflows

## MVC layer
All layers — security cuts across Controller (middleware chain), Service (RBAC, encryption), Model (sensitive fields), View (token storage).

## OWASP Top 10 checklist

### A01 — Broken Access Control
- [ ] Every route has `authenticate` middleware
- [ ] Every route has `requirePermission()` for the correct action
- [ ] Every Prisma query includes `organizationId: req.user.organizationId`
- [ ] Every approval endpoint checks `approverId !== requesterId`
- [ ] MANAGER role filters results to direct reports (`managerId: actor.id`)
- [ ] `req.body.role` never used to assign roles (role escalation prevention)

### A02 — Cryptographic Failures
- [ ] JWT secrets ≥ 32 characters (validated in `env.ts` with `.min(32)`)
- [ ] bcrypt rounds ≥ 12 for all password hashes
- [ ] AES-256-GCM for all sensitive fields (bank, SSN, salary) — field name ends in `Encrypted`
- [ ] HTTPS enforced via Nginx HTTP→HTTPS redirect
- [ ] `ENCRYPTION_KEY` is 64 hex chars (32 bytes)
- [ ] No sensitive data in localStorage or Redux state — tokens in httpOnly cookies only

### A03 — Injection
- [ ] All user input parsed through Zod `validateRequest()` middleware
- [ ] Prisma used for all DB queries — no raw SQL with user-supplied values
- [ ] File upload filenames sanitized before saving (no path traversal)
- [ ] No `eval()` or `Function(string)` with user-controlled input

### A04 — Insecure Design
- [ ] State machine enforces valid transitions only
- [ ] Terminal states are truly irreversible
- [ ] Optimistic lock pattern prevents race conditions on approvals

### A05 — Security Misconfiguration
- [ ] `helmet()` middleware active in all environments
- [ ] CORS origin is explicit — not `origin: '*'` for authenticated routes
- [ ] No `.env` committed to git
- [ ] No hardcoded credentials in source code
- [ ] Docker Postgres/Redis ports bound to `127.0.0.1`

### A06 — Vulnerable and Outdated Components
- [ ] `dependabot.yml` configured for weekly npm security patches
- [ ] No HIGH or CRITICAL vulnerabilities in `npm audit`

### A07 — Identification and Authentication Failures
- [ ] Refresh tokens stored in DB (revocable on logout)
- [ ] Refresh tokens use httpOnly, Secure, SameSite=strict cookies
- [ ] Auth routes rate limited: 50 req / 15 min
- [ ] Access token TTL: 15 minutes; Refresh token TTL: 7 days
- [ ] `JWT_SECRET` ≠ `JWT_REFRESH_SECRET` (different keys)

### A08 — Software and Data Integrity
- [ ] APK/AAB/IPA not committed to git
- [ ] `npm ci` used in CI (locked versions, not `npm install`)
- [ ] Build artifacts generated by CI pipeline, not manually

### A09 — Security Logging and Monitoring Failures
- [ ] Every create/update/delete has an audit log
- [ ] 4xx logged as warn, 5xx logged as error with stack trace (never exposed to client)
- [ ] No PII logged in plain text (passwords, tokens, account numbers)

### A10 — Server-Side Request Forgery
- [ ] No `fetch(req.body.url)` patterns without URL allowlist validation
